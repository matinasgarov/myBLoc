# CLAUDE.md — myblocate project context

## What this project is
A Next.js 16 / React 19 / TypeScript web app that scores business locations in Azerbaijan. User drops a Leaflet map pin → enters business type → gets a deterministic score + Groq AI analysis in Azerbaijani. See `documentation.md` for full details.

## Key files to know
| File | Purpose |
|---|---|
| `app/page.tsx` | State machine (`landing`/`map`/`input`/`loading`/`result`), all UI wiring |
| `components/Map.tsx` | Leaflet map — read the lifecycle section below before touching |
| `lib/overpass.ts` | OSM data fetch + competitor matching + `recognized` flag + new signals |
| `lib/az-competitors.ts` | AZ government dataset competitor lookup |
| `lib/score.ts` | 7-factor deterministic scoring (0–95 max, land use caps), returns `ScoreResult` |

## Map lifecycle — CRITICAL

### The ref pattern (do not break this)
`handlePinDrop` in `page.tsx` depends on `appState` via `useCallback`. If the map init effect depended on `onPinDrop` directly, every `appState` change would tear down and recreate the map mid-animation → `_leaflet_pos` crash.

Fix: the callback lives in a ref. Map init effect has `deps: []`. The ref is updated via a separate sync effect.

```ts
const dropRef = useRef(onPinDrop)
useEffect(() => { dropRef.current = onPinDrop }, [onPinDrop])
// init effect: deps []
map.on('click', (e) => dropRef.current(e.latlng.lat, e.latlng.lng))
```

### Reset remounts the map
`page.tsx` has `const [mapKey, setMapKey] = useState(0)`. `handleReset()` increments it. `<Map key={mapKey} ...>` causes React to unmount/remount the Map component cleanly. This is safe because no animation is in progress when user clicks reset.

**Do not replace this with `invalidateSize()` — it doesn't reload tiles when the container was sized wrong.**

## Scoring algorithm (7 factors — documentation.md has the table)
The score is calculated in `lib/score.ts` before the AI call. `calculateScore(ctx)` returns `ScoreResult = { score: number; factors: FactorResult[] }`. The AI only explains the score — it cannot change it. Max raw score is capped at 95; certain land uses (cemetery, industrial, etc.) have lower hard caps.

### Factor keys (`FactorKey`)
`competition` (22) · `footTraffic` (20) · `areaType` (13) · `urbanTier` (10) · `accessibility` (12) · `nearbyServices` (8) · `businessDensity` (10)

- **footTraffic** = `metroTierScore(metroRidership)` (0–12) + `roadScore(majorRoads)` (0–8). Note: bus stops go into **accessibility**, NOT footTraffic.
- **accessibility** = `busStopScore(busStops)` (0–7) + parking presence (0–5)

### Static data files (do not delete)
- `lib/metro-stations.ts` — auto-generated by `scripts/build-metro-data.mjs` from `datasets/pathways.csv` + `datasets/passenger_2025.csv`
- `lib/settlements.ts` — auto-generated by `scripts/build-settlements.mjs` from the AZ settlements GeoJSON in `datasets/`
- To regenerate: `node scripts/build-metro-data.mjs` / `node scripts/build-settlements.mjs`

## Business type matching
Both `overpass.ts` (`COMPETITOR_ALIASES`) and `az-competitors.ts` (`BUSINESS_TO_AZ_TYPES`) must be updated together when adding new business type keywords. Both currently include `'aptek'`/`'apteka'` for pharmacy. The `recognized` flag is `true` only if at least one alias list matches the input.

`lib/categories.ts` defines the `BusinessCategory` interface (`key`, `labelAz`, `labelEn`, `synonyms[]`, `pinned?`) and exports `BUSINESS_CATEGORIES` — ~60 entries covering retail, food, health, services, transport, agriculture, professional, and entertainment. The `pinned` flag marks the 3 default cards shown in the collapsed modal state.

## Groq grammar correction
`lib/groq.ts` post-processes all LLM output fields through `fixAzerbaijaniGrammar()` before returning to the client. The `AZ_CORRECTIONS` array contains regex replacements for known Azerbaijani grammar errors (e.g. `çoxluq →  çoxlu`). The prompt also includes explicit contradiction-prevention rules: if `competitors > 0`, the LLM is forbidden from writing "no competitor" pros; if `competitors = 0`, it is forbidden from writing "high competition" cons.

## Input validation
`/api/places` rejects business types that are < 2 chars, > 100 chars, or contain no consecutive letter sequence (`/[\p{L}]{2,}/u`). This handles Azerbaijani characters (ə, ş, ğ, etc.) correctly.

## Overpass fallback
Both Overpass queries use `.catch(() => ({ elements: [] }))` so the API never returns 500 when Overpass is down — it returns a degraded but valid PlacesContext.

## i18n
- `lib/az.ts` and `lib/en.ts` export string objects
- `lib/i18n.ts` exports `getStrings(lang)` and the `Lang` type
- Language persisted in `localStorage` under `'myblocate-lang'`
- When adding new UI strings, add to both `az.ts` and `en.ts`

## Tailwind v4
`globals.css` uses `@import "tailwindcss"` — NOT the v3 directives. Do not change this.

## CSP
`next.config.ts` defines Content-Security-Policy. OSM tiles require `img-src: https://*.tile.openstreetmap.org`. Any new external resource (fonts, images, APIs) needs to be added here or the browser will silently block it.

## Do not
- Add `react-leaflet` — the project uses raw Leaflet directly
- Change the map init effect deps from `[]` — this will cause the `_leaflet_pos` crash
- Add `.env` values to git — `GROQ_API_KEY` goes in `.env.local` only
- Use Tailwind v3 syntax (`@tailwind base` etc.)
- Replace the `key={mapKey}` reset pattern with `invalidateSize()` hacks
